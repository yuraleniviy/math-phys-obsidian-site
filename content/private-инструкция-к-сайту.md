# Локальное тестирование
В node.js cp на своей ветке: `npx quartz build --serve` в папке проекта само собой

# Публикация сайта
В node.js cp: `npx quartz sync` потом `git push origin main`

## Разработка сайта в локальном репозитории
1. Проверить, где ты сейчас `git branch`.  Должно быть `* main`. Если нет, то переключиться `git checkout main`.
2. Подтянуть изменения с GitHub `git pull origin main`. 
3. Проверить, что нет незакоммиченных изменений `git status`. Если изменения есть, то закоммитить 
```
git add .
git commit -m "название коммита"
```

# Приватность
- Файлы и папки, начинающиеся с private отфильтровываются в проводнике, при этом папка templates даже не пушится. 
- Если нужно заприватить файл отдельно, не меняя его название, нужно поставить галочку draft

# Правила заполнение контентом
1. Все задания и темы хранятся кучей в private-tasks-and-topics
2. Файлы, не содержащие properties, не индексируются и будет ERROR поэтому файлы предметов, тем и заданий нужно оформлять согласно шаблону. Удобно настроить вставку шаблона на Ctrl+Shift+T, но это к слову
3. Если в файл была вставлена картинка, а потом удалена, то нужно её найти и удалить из private-attachments, чтобы она не засоряла репозиторий. Удобно её найти и удалить через просмотр глобального графа
