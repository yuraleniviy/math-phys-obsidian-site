> #  ГЛАВНОЕ ПРАВИЛО: не пушить в main
> 
> 1. Открыть папку проекта в Node.js cp
> 2. Проверить, где ты сейчас 
> ```
> git branch
> ```
>   Должно быть `* main`. Если нет, то переключиться 
>   ```
>   git checkout main
>   ```
> 3. Проверить, что нет незакоммиченных изменений 
> ```
> git status
> ```
> 
>  Если изменения есть, то закоммитить 
>  
> ```
> git add .
> git commit -m "название коммита"
> ```
> 4. В итоге должно получиться `nothing to commit, working tree clean`.
> 5. Подтянуть изменения с GitHub 
> ```
> git pull origin main
> ```
> Убедиться, что ошибок не выдало.
> 
> 6. От обновленного main создать ветку 
> ```
> git checkout название-ветки
> ```
> Название ветки формируется как `буквенный_код_автора-текущая_дата`. 
> Пример: `YL-02-march`
>  
> 7. Открыть Obsidian и редактировать.
> 8. Проверить что получается через `npx quartz build --serve`.
> 9. Когда всё готово, то снова
> ```
> git add .
> git commit -m "название коммита"
> ```
> 10. Запушить В СВОЮ ВЕТКУ `git push origin название-ветки`.
> 11. Можно удалить ветку локально `git branch -D teacher/yuraleniviy`, но лучше дождаться, пока кто-то смерджит её в main
> 12. После того как ветка будет слита с main, можно её удалять и локально и глобально

# Приватность
- Файлы и папки, начинающиеся с private отфильтровываются в проводнике, при этом папка templates даже не пушится. 
- Если нужно заприватить файл отдельно, не меняя его название, нужно поставить галочку draft

# Правила заполнение контентом
1. Все задания и темы хранятся кучей в private-tasks-and-topics
2. Файлы, не содержащие properties, не индексируются и будет ERROR поэтому файлы предметов, тем и заданий нужно оформлять согласно шаблону. Удобно настроить вставку шаблона на Ctrl+Shift+T, но это к слову
3. Если в файл была вставлена картинка, а потом удалена, то нужно её найти и удалить из private-attachments, чтобы она не засоряла репозиторий. Удобно её найти и удалить через просмотр глобального графа
